# 08 真机验收清单

模拟器(Mock)能验证的项在 [README](README.md) 已覆盖;以下为真机手工验收,建议两台 iPhone(A 跑扫描侧,B 跑外设侧),或 B 用 LightBlue / nRF Connect 代替。

## 准备

- [ ] 两台 iPhone 安装本 App(真机构建需在 Xcode 里配置你的开发者签名)
- [ ] A 机:扫描页 ⚙️ 关闭"使用 Mock 数据源",首次会弹蓝牙权限,选允许

## 1. 权限与状态引导(异常 №1)

- [ ] A 机关闭蓝牙 → 扫描页出现橙色横幅"蓝牙已关闭"+ 恢复建议;开启后横幅消失
- [ ] 设置里关掉 App 蓝牙权限 → 横幅变"权限被拒绝"且有"打开设置"按钮

## 2. 扫描与广播解析

- [ ] B 机进"外设模式"页,显示"广播:进行中"
- [ ] A 机扫描,列表出现"WB-iPhone 心率带",能看到 RSSI 与广播服务"心率服务"
- [ ] 开"只扫心率设备"过滤,B 机仍在列表;附近其他杂设备消失
- [ ] B 机退出外设页(停止广播)→ A 机列表 6 秒内该设备消失(幽灵清理)
- [ ] 20 秒不操作,扫描自动停止

## 3. 连接与 GATT

- [ ] A 机点击设备 → 状态条变绿"已连接",服务列表出现 心率服务(0x2A37/0x2A38)
- [ ] B 机"订阅者"变 1
- [ ] 读取 0x2A38(传感器位置)返回 `02`
- [ ] RSSI 按钮点击刷新数值

## 4. 订阅与心率图表

- [ ] A 机进"实时心率"页:bpm 数字每秒跳动,曲线滚动,B 机"已推送"计数同步增长
- [ ] A 机返回(退出心率页)→ B 机订阅者不变但推帧继续/停止取决于详情页订阅状态;完全退出详情页 → B 机订阅者归 0

## 5. 断连与自动重连(异常 №4)

- [ ] 连接状态下把 B 机蓝牙关掉 → A 机状态条转橙色"断线,Xs 后第 N 次重连",日志页出现"意外断连"
- [ ] 10 秒内重开 B 机蓝牙与外设页 → A 机自动恢复"已连接",且服务/订阅自动重建(心率继续跳)
- [ ] 一直不恢复 → 第 6 次重试后状态条红色"已放弃",出现"重新连接"按钮
- [ ] A 机主动返回退出详情页再进 → 正常连接(主动断开不触发重连)

## 6. 后台广播降级(体验型,需一台安卓更佳)

- [ ] B 机外设页运行中按 Home 退后台 → A 机(iOS)重新扫描仍能发现但名字可能丢失;安卓 nRF Connect 大概率扫不到 → 印证 [05](05-后台模式与状态恢复.md)

## 7. 配对加密(需支持加密特征的外设,可选)

- [ ] 用 nRF Connect(安卓)建一个要求加密的特征,A 机订阅 → 弹系统配对框,拒绝 → 详情页错误条提示"要求加密链路";接受 → 操作成功

## 8. 私有协议(Mock 即可,真机同理)

- [ ] 协议控制台:PING → 150ms 内收到 PING 响应(seq 一致)
- [ ] "大包回显"→ 帧日志可见多条"↓ 收到分片"后组包出完整响应帧
- [ ] 日志页:全流程事件按时序完整可查
