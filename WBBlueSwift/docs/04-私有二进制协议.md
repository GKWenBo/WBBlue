# 04 私有二进制协议

标准 GATT 特征(心率、电池)只能覆盖标准场景;企业设备(手环、血糖仪、工业传感器、门锁)的业务命令几乎都走**私有二进制协议**:在一对"透传"特征上(本项目 FFF1:Write + Notify)自定义帧格式收发。这是企业 BLE 开发的核心技能。

实现:[PacketCodec.swift](../WBBlueSwift/Core/Protocol/PacketCodec.swift) · 单测:`PacketCodecTests.swift`(11 项)· 交互演示:设备详情 → 私有协议控制台。

## 帧结构

```
┌──────┬──────┬─────┬─────┬────────┬─────────┬────────┐
│ 0xA5 │ 0x5A │ cmd │ seq │ len(2) │ payload │ crc(2) │
└──────┴──────┴─────┴─────┴────────┴─────────┴────────┘
  帧头(同步字)  命令  序号   小端     0..512B    小端
```

- **帧头 A5 5A**:两字节同步字,用于流中定位帧边界。选不易在数据中自然出现的模式;
- **cmd**:命令码。本项目约定响应 = 请求 `cmd | 0x80`(0x01 ping → 0x81 pong),一眼分清上下行;
- **seq**:序号,请求/响应配对(并发命令不串台)、丢帧检测;
- **len**:payload 长度,小端。**必须设上限**(本项目 512)——损坏的 len 会让接收端傻等或撑爆内存;
- **crc**:CRC-16/CCITT-FALSE(poly 0x1021,init 0xFFFF),覆盖 cmd..payload。单测用标准校验向量 `"123456789" → 0x29B1` 钉死算法(CRC16 变体多,对接固件时先对齐这个向量)。

## 为什么必须做流式组包

BLE 特征是**字节管道,不保证业务帧边界**:

- **半包**:一帧 70B,MTU 净负载 20B,通知分 4 次到达;
- **粘包**:固件把两帧塞进一个通知,或发送缓冲合并;
- **坏帧**:链路误码、固件 bug。

`PacketAssembler` 状态机:

```
feed(chunk) → 追加缓冲 → 循环 {
    找帧头(之前的垃圾字节丢弃,保留末字节防拆一半的 0xA5)
    ├─ 不足 6B(读不到 len)→ 等下一块
    ├─ len 超限 → 报 .payloadTooLong,跳过帧头逐字节重同步
    ├─ 整帧未到齐 → 等下一块
    ├─ CRC 不符 → 报 .crcMismatch,跳过帧头逐字节重同步
    └─ 校验通过 → 吐出 .packet,移除整帧,继续(处理粘包)
}
```

**重同步策略**是鲁棒性关键:坏帧只丢 2 字节帧头再重找,而不是清空缓冲——避免一个误码把后面排队的好帧全部陪葬。

## 分包(发送方向)

```swift
let frame = PacketCodec.encode(packet)
let mtu = central.maximumWriteLength(id: deviceID, withResponse: true)
for chunk in PacketCodec.chunks(of: frame, mtuPayload: mtu) {
    try await central.writeValue(..., data: chunk, withResponse: true)
}
```

有响应写自带流控(一来一回);改免响应写提速时必须处理发送窗口(见 [03-异常处理手册](03-异常处理手册.md) №6)。

控制台页的"大包回显"按钮发 60B payload,Mock 固件按 20B 分片回传,可在帧日志里直观看到 分片到达 → 组包 → 解析 全过程。

## 与 OTA 的关系

设备固件升级(OTA)就是私有协议的极致应用,协议模式完全同构:

1. 握手:查版本(cmd 0x03)→ 判断是否需要升级;
2. 开始:发固件元信息(大小、总 CRC、块大小);
3. 传输:固件按块分包下行(免响应写提吞吐),每块带块号(seq 放大版)与块 CRC,设备按块回 ACK/NAK,错块重传;
4. 校验:全部传完设备校验总 CRC,回成功/失败;
5. 激活:设备重启进新固件,App 重连验证版本号。

工程要点:断点续传(记录已确认块号)、传输中断连的恢复(重连后从断点继续)、低电量拒绝升级、双分区(A/B)防变砖。这些都是 №4 自动重连 + 本篇协议层的组合应用。
